# コミット作成

gitのコミットを作成する。

## 手順

1. `git status` と `git diff`（ステージ済み・未ステージ両方）を実行して変更内容を確認する
2. 変更内容に基づいてコミットメッセージを作成する
3. 関連する変更ファイルを `git add` でステージングする
4. コミットを実行する
5. `git status` で結果を確認する

## コミットメッセージのフォーマット

```
{prefix}: {変更内容の短い概要文}

- {詳細1}
- {詳細2}
...（1〜5行の箇条書き）
```

## prefix一覧

| Prefix | 説明 |
|--------|------|
| `feat` | 新しい機能の実装 |
| `fix` | 不具合修正 |
| `docs` | ドキュメントの変更 |
| `style` | コードフォーマットの修正 |
| `refactor` | リファクタリング |
| `perf` | パフォーマンス・チューニング |
| `test` | テストコードの追加・変更 |
| `chore` | ビルドプロセス、ツール、ライブラリの変更 |

## ルール

- コミットメッセージは日本語で書くこと
- 概要文は簡潔でわかりやすくすること
- Claudeの署名（Co-Authored-Byなど）は付けないこと
- コミットメッセージはHEREDOCで渡すこと
- .envやcredentials.jsonなど秘密情報を含むファイルはコミットしないこと
