# コードレビュー

現在のブランチのコード変更をレビューする。

## 手順

1. `git branch --show-current` で現在のブランチ名を確認する
2. `git diff main...HEAD` で変更差分を取得する
3. 変更差分がない場合はエラーメッセージを表示して終了する
4. 以下の6つのエージェントをTaskツールで**並列に**起動する
5. 全エージェントの結果を待機し、結果をまとめてユーザーに報告する

## 起動するエージェント

Taskツールを使用して、以下の6つのエージェントを**1つのメッセージで並列に**起動する。
各エージェントには `git diff main...HEAD` の結果を渡す。

| エージェント | ファイル | 観点 |
|-------------|----------|------|
| Rustベストプラクティス | `.claude/agents/review-rust-best-practices.md` | 慣用的な書き方、所有権・借用、エラーハンドリング |
| アンチパターン | `.claude/agents/review-anti-patterns.md` | 不要なクローン、過度な結合、神オブジェクト |
| パフォーマンス | `.claude/agents/review-performance.md` | 不必要なアロケーション、非効率なイテレーション |
| セキュリティ | `.claude/agents/review-security.md` | unsafeの使用、パニックの可能性、入力バリデーション |
| 可読性・保守性 | `.claude/agents/review-readability.md` | 命名、責務分離、コードの重複 |
| エラーハンドリング | `.claude/agents/review-error-handling.md` | anyhowの使用、context追加、unwrap禁止ケース |

## 結果の報告形式

全エージェントの結果を以下の形式でまとめる:

```markdown
## レビュー結果

### Rustベストプラクティス
[エージェントの結果]

### アンチパターン
[エージェントの結果]

### パフォーマンス
[エージェントの結果]

### セキュリティ
[エージェントの結果]

### 可読性・保守性
[エージェントの結果]

### エラーハンドリング
[エージェントの結果]

## 総評
[全体的な評価とまとめ]
```

## ルール

- レビュー結果は日本語で報告すること
- 6つのエージェントは必ず並列で起動すること
- 重大な問題は明確にマークすること
