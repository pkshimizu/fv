# GitHub Issue作成

GitHub Issueを作成する。

ユーザーの要望: $ARGUMENTS

## 手順

### 1. 要望の分析

ユーザーの要望を分析し、INVEST原則に基づいて適切な粒度のissueに分割する。

- **I (Independent)**: 各issueが他のissueに依存せず独立して完了できること
- **N (Negotiable)**: 実装方法に柔軟性があること
- **V (Valuable)**: 各issueが単体で価値を提供すること
- **E (Estimatable)**: 作業量が見積もり可能な大きさであること
- **S (Small)**: 1つのissueが大きくなりすぎないこと
- **T (Testable)**: 完了条件が明確でテスト可能であること

要望が大きい場合は複数のissueに分割する。小さい場合は1つのissueで構わない。

### 2. テンプレートの選択

各issueの内容に応じて、`.github/ISSUE_TEMPLATE/` から適切なテンプレートを選択する。

| テンプレート | 用途 |
|--------------|------|
| `feat.yml` | 新しい機能の実装 |
| `fix.yml` | 不具合修正 |
| `docs.yml` | ドキュメントの変更 |
| `style.yml` | コードフォーマットの修正 |
| `refactor.yml` | リファクタリング |
| `perf.yml` | パフォーマンス・チューニング |
| `test.yml` | テストコードの追加・変更 |
| `chore.yml` | ビルドプロセス、ツール、ライブラリの変更 |

### 3. ユーザーへの確認

issueを作成する前に、以下の内容をユーザーに提示して確認を取る。

- 作成予定のissue一覧（タイトル、分類、概要）
- 分割の妥当性
- 不明点や追加で確認が必要な詳細

### 4. issueの作成

ユーザーの承認を得たら、`gh` コマンドでissueを作成する。

issueのタイトルは `<分類>: <内容>` の形式にする（例: `feat: ユーザー認証機能の追加`）。

テンプレートの各フィールドに対応する内容を本文に含める。本文はマークダウン形式で、テンプレートの見出しに合わせて記述する。

```bash
gh issue create --title "<分類>: <タイトル>" --label "<分類>" --body "<本文>"
```

複数のissueを作成する場合は、順番に作成し、各issueのURLを記録する。

### 5. 結果の報告

作成したすべてのissueのURLをユーザーに報告する。

## ルール

- issueのタイトル、本文は日本語で書くこと
- Claudeの署名（Co-Authored-Byなど）は付けないこと
