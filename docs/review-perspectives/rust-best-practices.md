# Rust ベストプラクティス

Rust の慣用的な書き方に従っているかをレビューする観点。

## レビュー観点

- Rust のイディオムに従っているか
- 標準ライブラリを適切に使っているか
- 所有権・借用を正しく使えているか
- エラーハンドリングが適切か（`Result` / `Option` の使用）
- `?` 演算子を適切に使っているか
- イテレータを効果的に活用しているか
- パターンマッチングを適切に使っているか
- clippy 警告に該当するパターンがないか
- `derive` マクロを適切に使っているか

## 重点

- 具体的なコード例を添えて改善案を示す。
