# セキュリティ

安全性に関わる問題を検出する観点。

## レビュー観点

- `unsafe` コードの適切な使用と安全性の保証
- 入力のバリデーション
- パニックの可能性（`unwrap` / `expect` の乱用）
- 機密情報の漏洩リスク（ログ出力、エラーメッセージ）
- 信頼できない入力の処理
- バッファオーバーフローの可能性
- 整数オーバーフローの可能性
- パス・トラバーサル攻撃の可能性
- コマンドインジェクションの可能性
- 適切なエラーハンドリング（情報漏洩を避ける）
- 時間ベースの攻撃への耐性

## 重点

- 重大なセキュリティ問題は必ず「重要度: 高」としてマークする。
- 具体的な攻撃シナリオがあれば説明する。
