# エラーハンドリング方針への準拠

本プロジェクトのエラーハンドリング方針に準拠しているかをレビューする観点。

## エラーハンドリング方針

### 基本方針

- **エラー型**: `anyhow` クレートを使用する。
- **エラーログ**: `tracing` クレートで `~/.fv/error.log` に出力する。
- **ユーザー向けメッセージ**: UI 層で定義する。

### エラー情報の分離

| 対象 | 内容 | 出力先 |
|------|------|--------|
| 開発者向け | エラーチェーン、スタックトレース、コンテキスト | tracing の設定に依存 |
| ユーザー向け | 簡潔なメッセージ | UI ステータスバー |

### ライブラリ層（UI 層以外）の実装ルール

1. 戻り値の型は `anyhow::Result<T>` を使用する。
2. `?` 演算子でエラーを伝播する。
3. ライブラリ層ではログ出力しない（UI 層に委譲）。

### `unwrap()` / `expect()` の使用ルール

**許可するケース**

- テストコード
- 起動時の不変条件（ホームディレクトリの取得など）
- 論理的にパニックが適切な場合

**禁止するケース**

- ユーザー操作に依存する処理
- ファイル I/O
- 外部リソースへのアクセス
- ネットワーク操作

## レビュー観点

- `anyhow::Result<T>` が戻り値の型として使用されているか
- `?` 演算子でエラーが適切に伝播されているか
- ライブラリ層でログ出力（`tracing::error!` 等）をしていないか
- `unwrap()` / `expect()` が禁止ケースで使用されていないか
- 独自エラー型を定義せず `anyhow` を使用しているか
