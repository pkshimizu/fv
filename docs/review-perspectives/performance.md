# パフォーマンス

実行効率に関わる問題を検出する観点。

## レビュー観点

- 不必要なヒープアロケーション
- 非効率なイテレーション（`collect` 後の再イテレーションなど）
- ロック競合の可能性（`Mutex` / `RwLock`）
- 不要な計算や重複処理
- 大きなデータのコピー（値渡し vs 参照渡し）
- 文字列操作の効率性（`String` 結合 vs `format!`）
- ベクタの事前容量確保（`with_capacity`）
- 不要な中間コレクション生成
- ホットパスでの重い処理
- 適切なデータ構造の選択（`HashMap` vs `BTreeMap` など）

## 重点

- 具体的なコード例を添えて改善案を示す。
