# ファイルシステム表記の集約ルール

パーミッション（rwx / mode）・サイズ・日時など、ファイルシステム情報の**表示・変換ロジックは
`fs` 層に集約**する。component などの上位レイヤで同じ変換を再実装しない。

- rwx 文字列・mode の表記は `fs/permissions.rs` の `VPermissions` を使う
  （`to_rwx_string()` / `from_mode()` / `rwx_bits()`）。component で `0o400` などのマスク表や
  rwx 生成ロジックを別に持たない。
- サイズ・日時の整形は `VFileMetadata` / `VFileTime` 側のメソッドを使う。
- 編集 UI 固有の関心（カーソル位置とビットの対応など）はビュー側に置いてよいが、その場合も
  **マスクや文字の定義は `fs` 層の単一の真実源を参照**する（`VPermissions::rwx_bits()` など）。

理由: 同じ変換が 2 箇所にあると、片方だけ変更してビューと実体・他画面の表示がずれる。
表示フォーマットの完全一致が要る箇所では、レイヤが違っても別実装にしない。
